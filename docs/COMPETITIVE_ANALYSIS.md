# GEO Optimizer 竞品分析与优化审查报告

> 本报告由专业代码与产品审查团队生成，基于 GitHub 开源项目深度调研。

## 一、竞品全景图

### 1.1 核心竞品对比

| 项目 | 核心能力 | 与本项目重叠度 | 关键差异 |
|------|---------|--------------|---------|
| **GetCito** | AI 搜索品牌引用监控 + 竞品对比 | ★★★★☆ | 仅监控层，无内容生成与分发 |
| **AutoGEO** | 规则提取 + RL 优化，50% 可见性提升 | ★★★★☆ | 学术级，无产品化 UI |
| **Gego** | 多 LLM GEO 追踪 + 关键词提取 | ★★★☆☆ | CLI 工具，无中文平台支持 |
| **GEO-Analyzer** | Claude 驱动语义分析评分 | ★★★☆☆ | 单点分析，无闭环流程 |
| **GEO-optim** | 学术 GEO-Bench 基准框架 | ★★☆☆☆ | 纯研究框架，非产品 |

### 1.2 本项目核心竞争力

```
监控层（GetCito / aperture / llm-brand-tracker）
  ↓
审计层（aeo-audit / seo-aeo-geo-analyzer）
  ↓
优化策略层（brightdata/geo-ai-agent / aeo-box）
  ↓
内容生成层（claude-blog / AEOContentCreation）   ← 本项目覆盖
  ↓
分发执行层（SocialSpread_Workflow — 仅有且不成熟）  ← 本项目覆盖
  ↓
效果监控 + 反馈闭环                              ← 本项目独有
```

**核心结论**：本项目 = 监控 + 审计 + 策略 + 生成 + 分发全链路打通，且聚焦中文平台生态。这个组合在全 GitHub 范围内没有任何一个仓库做到。

## 二、优化审查发现与已实施改进

### 2.1 GEO 评分体系升级 ✅ 已实施

**问题**：原评分仅 4 维度（Citability 30%, ClaimDensity 25%, Extractability 25%, Readability 20%），缺少权威性引用、平台适配度等关键维度。

**改进**：升级为 6 维度评分模型（基于普林斯顿 KDD 2024 论文增强版）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 权威性引用 (Authority Citation) | 20% | 专家/机构引用、学术背书 |
| 结构化数据 (Structured Data) | 15% | FAQ/表格/列表等结构化元素 |
| 关键词覆盖度 (Keyword Coverage) | 20% | 关键词 + 长尾词覆盖率 |
| 可信度信号 (Credibility Signals) | 15% | 具体数据、案例、来源标注 |
| 平台适配度 (Platform Fitness) | 15% | 内容长度/风格匹配目标平台 |
| 更新时效性 (Freshness) | 15% | 时间信号和信息新鲜度 |

### 2.2 平台内容调性差异化配置 ✅ 已实施

**问题**：原 `template_manager.py` 仅有 4 个平台的简单规则（name/length/style），缺乏深度调性配置。

**改进**：新增 `PLATFORM_STYLES` 丰富元数据层：

| 平台 | 语气调性 | 字数范围 | 内容结构 | Emoji 密度 |
|------|---------|---------|---------|-----------|
| 公众号 | 深度专业、有温度感 | 1200-2200 | 标题→引言→正文→总结→引导关注 | low |
| 小红书 | 闺蜜聊天、真实分享感 | 300-800 | 封面标题→痛点共鸣→干货分享→互动引导 | high |
| 知乎 | 理性客观、数据支撑 | 800-1500 | 观点→论据→案例→引用→总结 | none |
| 短视频 | 口语化、节奏感强 | 100-500 | Hook→痛点→解决方案→CTA | low |
| 百家号 | 客观权威、信息密集 | 1000-2000 | 新闻标题→导语→正文→总结→延伸阅读 | none |

### 2.3 行业可插拔配置系统 ✅ 已实施

**问题**：系统无行业配置概念，无法针对特定垂直领域优化内容。

**改进**：新增 `industry_config.py` 可插拔行业配置包系统：

- **情感咨询/婚姻修复**（MVP 切入点）— 含 15 个种子关键词、10 个禁用词、5 平台 hints
- **教育培训** — 含种子关键词、禁用词、3 平台 hints
- **电商零售** — 含种子关键词、禁用词、2 平台 hints

每个行业包含：
- `seed_keywords`：行业种子关键词库
- `banned_words`：行业禁用词汇
- `default_tone`：默认内容调性
- `platform_hints`：各平台开场白 / CTA / 标签建议

### 2.4 API 层扩展 ✅ 已实施

新增 `/api/v1/industries` 端点组：
- `GET /industries` — 列出所有行业配置
- `GET /industries/{id}` — 获取行业详细配置
- `GET /industries/{id}/keywords` — 获取行业种子关键词
- `GET /industries/{id}/platform-hints/{platform}` — 获取平台特定建议
- `GET /industries/platforms/styles` — 获取所有平台调性元数据

### 2.5 CI/CD 自动化 ✅ 已实施

新增 GitHub Actions 工作流 (`.github/workflows/ci.yml`)：
- 后端：Python 3.11 + pip + pytest 自动测试
- 前端：Node.js 20 + npm + TypeScript 检查 + Vite 构建

### 2.6 测试覆盖 ✅ 已实施

新增 53 个单元测试覆盖：
- `test_geo_scorer.py` — GEO 评分引擎全维度测试（31 个测试）
- `test_template_manager.py` — 平台模板配置测试（12 个测试）
- `test_industry_config.py` — 行业配置系统测试（10 个测试）
